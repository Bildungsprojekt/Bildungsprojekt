# Richtlinien für Beiträge zum Bildungsprojekt
Schön, dass du dabei bist! Egal ob du Texte für das E-Book hast oder Code für die Bildungsplattform beisteuern willst, hier erfährst du genau, wie es geht.

## ⚖️ Die rechtliche Grundregel (Pflicht!)
Sobald du hier etwas einreichst (über ein Issue Formular oder einen Pull Request), bestätigst du automatisch verbindlich:
* **Du bist der alleinige Rechteinhaber** oder hast die nachweisbare Einwilligung.
* Du stellst deinen Beitrag unter die freie Lizenz **CC BY SA 4.0** (weltweit und unbefristet).
* Du haftest selbst, falls dein Beitrag die Rechte anderer verletzt.

---

## 🎯 Weg A: Ich habe eine Idee oder einen Text (Kein Code)
Du musst nicht programmieren können, um das Projekt besser zu machen! 

1. Gehe oben auf dieser Seite auf den Reiter **Issues**.
2. Klicke auf den grünen Button **New issue**.
3. Wähle die Vorlage **Neuer Beitrag oder Ziel** und klicke auf *Get started*.
4. Fülle das Formular aus. Wir fragen dort deinen Text, die rechtliche Bestätigung und deine E Mail Adresse ab.
5. Klicke auf *Submit new issue*. Fertig!

*Tipp: Wenn dir GitHub zu technisch ist, kannst du auch einfach direkt drüben auf Wattpad in den Kommentaren deine Ideen teilen!*

---

## 👨‍💻 Weg B: Ich bin Entwickler und will Code beisteuern
Du willst an der Architektur der App schrauben? Perfekt.

### 1. Eine Aufgabe übernehmen
Schau in die bestehenden **Issues**. Wenn du ein Problem beheben oder ein Ziel umsetzen willst, klicke auf **New issue**, wähle die Vorlage **Aufgabe übernehmen oder Update posten** und sag uns dort kurz Bescheid. So verhindern wir, dass zwei Leute am selben Code arbeiten.

### 2. Code schreiben und einreichen (Pull Request)
* **Branch Namen:** Nutze `feature/[name]` für neue Funktionen oder `fix/[name]` für Fehlerbehebungen.
* **Pull Request erstellen:** Wenn dein Code fertig ist, sende einen Pull Request an unseren Branch `prototyp`.
* **Prüfung:** Beschreibe im Pull Request kurz, was dein Code macht und wie wir ihn lokal testen können. Pack gerne einen Screenshot oder ein kurzes Video dazu.

Wir testen deinen Code dann: Lädt die App? Funktioniert die Änderung? Erst nach einem erfolgreichen Test wird dein Beitrag fest integriert.
